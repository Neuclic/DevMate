# Skills Workflow

Skills are reusable notes that capture a successful delivery pattern.

## What To Save

- Save a skill when the implementation pattern is likely to repeat.
- Focus on tasks such as creating a static site, wiring an API service, or packaging Docker delivery.
- Keep the summary short and the steps explicit enough to follow without rereading the full session.

## What A Good Skill Contains

- A clear skill name that maps to the repeated task.
- A short summary explaining when to use it.
- A small ordered set of execution steps.
- Optional tags or keywords that help retrieval.

## Reuse Strategy

- Search local skills before starting a new plan.
- Inject the best-matching skill summaries into the planning context.
- Save a new skill only when the outcome is stable enough to repeat.
