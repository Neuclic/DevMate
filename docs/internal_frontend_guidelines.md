# 内部前端规范样例

这份文档用于给后续 RAG 演示提供一个可检索样本。

## 页面结构建议

- 首页优先展示核心价值和主入口操作
- 地图类页面优先展示筛选器、位置状态、结果总数
- 移动端优先考虑单列布局

## 视觉建议

- 不使用过于廉价的默认紫色渐变
- 页面首屏要有明确标题和行动按钮
- 地图与列表要形成主次关系

## 工程建议

- HTML、CSS、JavaScript 分文件管理
- 静态页面也要保留可扩展的目录结构
- 对地图 SDK 做单独封装，避免逻辑散落在入口文件中

## 可访问性建议

- 按钮必须有清晰文案
- 地图交互之外要有列表兜底
- 颜色不是唯一的信息表达方式
