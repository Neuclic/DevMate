# Build Static Site

## Summary

Use this skill when the user asks for a small static website with a clear file split and lightweight client-side behavior.

## Steps

1. Confirm the target page sections, responsive behavior, and any third-party browser SDK needs.
2. Split the output into at least HTML, CSS, and JavaScript files with a clear asset boundary.
3. Keep map integrations behind a small wrapper so API keys and provider calls stay isolated.
4. Add one quick verification path describing how to open or serve the generated site locally.
