# Webapp Testing

## Summary

Use this skill when a generated website or UI needs a quick validation pass before you trust the output.

## Steps

1. Check initial render in a desktop viewport and confirm the main page structure loads without missing assets.
2. Recheck the same page in a narrow mobile viewport and look for overflow, clipped text, and unusable controls.
3. Interact with the primary UI path and record obvious regressions before making deeper changes.
4. Summarize findings as concrete issues, not vague impressions.
